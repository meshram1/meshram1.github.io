/ 05

Notes on the model

This is a didactic simulator — the goal is to internalise how virtual topology (threads, blocks, grid) and physical topology (warps, SMs, memory) compose, not to predict CUDA wall-clock down to the microsecond.

Time = total FLOP ÷ (peak FLOP·s−1 × occupancy × warp efficiency × coalescing × memory pattern) · cost = time × hourly rate.

Warp efficiency drops when threads / block isn't a multiple of 32 — the last warp runs with idle lanes. Coalescing rewards block dim X ≈ 32 because a warp of 32 contiguous threads should read 32 contiguous floats. Occupancy is bounded by the SM's max threads (2048) and max blocks (32) per SM. Memory pattern scales the achievable fraction of peak throughput — shared-memory tiling is the biggest single lever.

Hardware blocks are placeholders — SM counts, warps/SM, FLOPs, bandwidth, and $/hr are loosely inspired by real generations but named generically. Swap in your own numbers in poisson-game.js if you want to model a specific card.

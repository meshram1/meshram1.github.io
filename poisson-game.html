/ 02

The math, in a tile

/ continuous PDE
∇2u = f

Poisson's equation. Find a field u whose curvature equals a given source f. Shows up everywhere — pressure projection in incompressible flow, electrostatics, gravity, heat at steady state.

/ central differences on a grid (spacing h)
∂2u/∂x2 ≈ ( ui+1,j − 2ui,j + ui−1,j ) / h2
∂2u/∂y2 ≈ ( ui,j+1 − 2ui,j + ui,j−1 ) / h2

Discretize. Replace each second derivative with the difference of its two neighbours. The PDE becomes one algebraic equation per grid cell.

/ jacobi iteration — the inner loop you're optimizing
ui,jn+1 = 14 ( ui+1,jn + ui−1,jn + ui,j+1n + ui,j−1n − h2fi,j )

One iteration, one cell: read 4 neighbours from the old field, average them (correcting for the source), and write the new value. Now repeat — for every cell, 10,000 times. That's the workload your GPU mapping has to chew through.

the 5-point stencil

Per cell update: 4 reads + 1 write + ~7 floating-point ops. The whole solver is just this, repeated 1,048,576 cells × 10,000 iterations.

why memory pattern is the lever each interior cell is touched 5 times per iteration — once as the centre, four more times as a neighbour to its neighbours
no ping-pong

One array. While thread A reads its right-neighbour, thread B is already writing into that same cell. You get a mix of old and new values — the math is wrong.

+ ping-pong

Two arrays, un and un+1. Read from one, write into the other, swap. Every neighbour read is guaranteed to be the consistent old value.

no shared mem

Each cell is fetched 5 times from DRAM per iteration. DRAM is hundreds of cycles away. Compute sits idle waiting on memory — you're memory bound.

+ shared mem

Each block loads its tile + a 1-cell halo into the SM's on-chip scratchpad once. The 5 reuses now happen at L1/SRAM speed — roughly 10× faster than going back to DRAM.

Choose physical hardware

/ 06

Notes on the model

This is a didactic simulator — the goal is to internalise how virtual topology (threads, blocks, grid) and physical topology (warps, SMs, memory) compose, not to predict CUDA wall-clock down to the microsecond.

Time = total FLOP ÷ (peak FLOP·s−1 × occupancy × warp efficiency × coalescing × memory pattern) · cost = time × hourly rate.

Warp efficiency drops when threads / block isn't a multiple of 32 — the last warp runs with idle lanes. Coalescing rewards block dim X ≈ 32 because a warp of 32 contiguous threads should read 32 contiguous floats. Occupancy is bounded by the SM's max threads (2048) and max blocks (32) per SM. Memory pattern scales the achievable fraction of peak throughput — shared-memory tiling is the biggest single lever.

Hardware blocks are placeholders — SM counts, warps/SM, FLOPs, bandwidth, and $/hr are loosely inspired by real generations but named generically. Swap in your own numbers in poisson-game.js if you want to model a specific card.
